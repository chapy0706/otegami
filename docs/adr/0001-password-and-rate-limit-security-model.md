# ADR-0001: 短いパスワードを許容し、URLの推測困難性とレート制限で守る

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

otegami は手打ちでの閲覧を前提とする。長いパスワードは入力負担が大きく、自分宛ての一時的な持ち出しという用途に対して過剰である。

一方で、短いパスワード単体は総当たりに弱い。半角英数4文字は 62^4 ≈ 1,470万通りで、無防備なら短時間で破られる領域にある。「短いから安全」という発想は誤りで、パスワードの長さそのものは攻撃を防がない。守りの責務を別の層へ移す必要がある。

## 決定

- 閲覧パスワードは半角英数1〜4文字に制限する。短さは手打ちの利便のための割り切りと位置づける。
- アクセス制御の主たる鍵は推測困難な slug とする。slug を知らなければ閲覧の入口に到達できない。
- 短いパスワードへの総当たりは、同一IP・同一slug に対する閲覧試行のレート制限で実質的に封じる。
- パスワードは Argon2 でハッシュ化して保存し、平文は持たない。
- 閲覧失敗は「見つからない」と「パスワード違い」を区別しない単一エラーとし、ノートの存在を推し量らせない。

## 結果

- 利点: 手打ちの軽さを保ちつつ、現実的な総当たりを防げる。鍵を一枚に依存しない多層防御になる。
- トレードオフ: slug が漏れた相手に対しては軽い障壁しか課せない。秘匿性の主役は slug の管理であり、URLの取り扱いに注意が要る。
- 前提条件: レート制限は本モデルが成立するための必須要素である。これを欠くと短いパスワードの妥当性が崩れるため、実装で必ず備える。

## 考慮した代替案

- パスワードを長く必須化する: 手打ち用途に反するため却下。
- パスワードを廃し slug のみで守る: 共有経路に slug が残ると即座に閲覧されてしまう。軽い二段目として残す。
