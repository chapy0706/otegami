# ADR-0004: 作成導線の認証を Cloudflare Access に委譲する

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

作成できるのは運用者だけにしたい。一方で、閲覧はURLとパスワードを知る者なら誰でも可、という非対称な要件がある。

otegami は既に Cloudflare Tunnel 経由で配信される。アプリ内に OIDC 検証(Keycloak とトークン検証)を実装する選択肢もあるが、一人運用のための認証コードとしては重い。また「運び手に不要な記憶を持たせない」という方針上、アプリが身元情報を抱えない方が望ましい。

## 決定

- 作成系エンドポイントのみ Cloudflare Access で保護し、運用者本人に限定する。
- 閲覧経路 `/n/<slug>` は公開のままとし、URL とパスワードで制御する。
- アプリ内には認証・セッション・ユーザーという概念を持ち込まない。

## 結果

- 利点: アプリ側の認証コードがゼロになり、攻撃面と保守対象が減る。既存の Cloudflare 基盤にそのまま乗る。
- トレードオフ: 作成導線の可用性とアクセス制御が Cloudflare に依存する。設定ミスは作成不能、あるいは意図しない開放に直結するため、Access の設定は構成として管理する。
- 移植性: 別基盤へ移す場合、作成導線の保護を自前(リバースプロキシ認証や OIDC)で組み直す必要がある。その時点で本ADRを置換する。

## 考慮した代替案

- Keycloak + Auth.js もしくはアプリ内 OIDC: 自前で身元を扱えるが、一人運用には実装・運用コストが見合わない。
- 作成もパスワードで保護する: 作成と閲覧の権限差を表現できず却下。
